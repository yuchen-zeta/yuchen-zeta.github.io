BTC-密码学原理

发表于2021-06-16更新于2021-06-16字数统计745
  • 比特币是一种经典的加密货币crypto-currency
    基于区块链技术的比特币的安全性主要依赖于共识机制与密码学的保证
  • 比特币主要用到了密码学中的两个功能：1.哈希 2.签名

哈希 hash

密码学中用到的哈希函数被称为 cryptographic hash function，有以下重要性质：

collision resistance:

  • 给定一个hash函数，存在x和y，x≠y，使得hash(x) = hash(y)
    即两个不同的输入，对应了同一个输出，这就称为哈希碰撞。
  • 注意若输入空间大于输出空间，哈希碰撞是不可避免的，但是可能性微乎其微，而且无法人为制造。
    目前也不存在一个hash函数可从数学上证明其具有collision resistance的性质，这是根据实践经验得来的。

hiding:

  • hiding 指 hash函数的计算过程是单向的，不可逆的( 即从hash(x)无法推出x )
  • hiding性质的前提是输入空间足够大，分布比较均匀(太小可以蛮力求解brute-force)。
    如果不够大，可在x后拼接一个随机数如hash( x || nonce )

实现 digital commitment ( 数据保证 ):

  • 把预测结果作为输入x，提前公布hash(x)，这样hiding 让人们知道哈希值而不知道预测内容，等结果出现后，公布预测结果x，又由collision resistance 保证该结果是不可篡改的，这样事预测结果可以使人信服。

puzzle friendly:

  • 该性质指哈希值事先是不可预测的，挖矿没有捷径。
  • 两个极其相似的输入取哈希后会截然不同，所以无法知道哪些输入值更容易算出某个哈希值，只能一个一个试。
  • 该性质保证了比特币系统中，工作量证明(proof of work)机制可以运行下去。
  • 比特币中用的哈希函数是**SHA-256( secure hash algorithm )**，以上三个性质它都满足。

签名 signature

  • 比特币系统中开设账户是在本地创立的，本地创立一个公钥-私钥对(private key, public key)
    这就是一个账户(此处应用的是密码学的非对称加密体系)
  • 公私钥体系用签名来提供和验证身份信息，发布交易时，用自己的私钥签名，其他人通过公钥验证，具体如下
    • 交易发起者 对一笔交易取摘要 ，将摘要用私钥进行加密(即签名)，将签名信息附在交易信息上，矿工用该交易发起者的公钥解密签名信息得到摘要信息，并对该交易做摘要，对比这两者是否相等(即验证签名)
