一、OSI与TCP-IP 5层协议

分层模型

分层思想

  • 将复杂的流程分解为几个功能相对单一的子过程

    • 整个流程更加清晰，复杂问题简单化
    • 更容易发现问题并针对性地解决问题
    • 同层使用相同协议，下层为上层提供服务

  • 使用网络的未必知道数据的传输过程

OSI七层模型

  • OSI（Open System interconnection）
    是开放的通信系统互联参考模型
  • 历史：1947 年，ISO 组织发布了 OSI 参考模型

—OSI 七层模型—

  • TCP/IP 5层协议簇/协议栈：会话层，表示层，应用层三层合并成为应用层，最后为五层协议
  • 这两个同时诞生，OSI 先有模型，TCP 先有协议
  • 四层模型：把 数据链路层 和 物理层 合并为网络接口层
  • 特点：上层对下层兼容，处于上层的设备一定可以识别所有下层的信息

OSI模型 vs TCP/IP模型

数据传输过程

数据封装过程

假设要发送“Hello”这个数据，首先是封包

1.通过应用层，在软件中生成，进入传输层

2.传输层对数据进行 封装 ，进入网络层

  • 传输层 完成进程与进程之间的通信
  • 封装：给数据加上 TCP/UDP 头
    在加头之前判断数据是从应用层中的哪个软件生成的
    然后在数据中加上原端口号和目标端口号(服务器端口号)
  • 封装完后整个数据为 数据段
  • 端口号是应用层的
    UDP：速度快，但安全性不高，不可靠传输(双方不建立连接)，像写信
    TCP ：速度不快，但安全性高，可靠传输(双方要建立连接)，像打电话

3.网络层对数据再加上 IP包头 ，进入数据链路层

  • 网络层 完成点到点的通信 (行话，”点”指电脑)
  • 加上 IP 包头(包含源IP和目标IP)，加上后整个数据为 IP包/报文

4.数据链路层（网卡）给数据加上 MAC子层 和 FCS ，进入物理层（网线）

  • 交换机工作在 数据链路层，意味着交换机没有上层的功能
  • MAC子层：简称帧头，源 MAC 地址和目标 MAC 地址
    FCS：帧校验序列，简称帧尾，4个字节，循环校验算法 CRC（类似hash）中的值，加上后整个数据为 帧

5.物理层(网线) 将 帧 用数字信号(比特流)传输出去，经过交换机、路由器、服务器、防火墙等一系列设备到达 对方电脑网卡

数据解封装过程

6.数据链路层（网卡）检查 帧 的 MAC子层 和 FCS，检查MAC子层的目标MAC地址 与 该网卡的地址 是否吻合，吻合则解封装，不吻合就扔掉

7.网络层接着校验IP包/报文，检查目标IP是不是本机，是则解包，不是就扔掉

8.传输层对数据段解析，由TCP/UDP头里的目标端口号 将信息传给应用层的正确进程

要点总结

  • 各层的 数据名称 及 对应设备：

    1. 数据/PDU 应用层 PC/防火墙 五层
    2. 数据段/段 Fragment 传输层 防火墙 四层
    3. 报文/包/IP包 packet 网络层 路由器 三层
    4. 帧 Franme 数据链路层 交换机 网卡 二层
    5. 比特 bit 物理层 网线 一层
      • 8 bit = 1 Byte字节
      • 所谓的百兆宽带 (* ￣︿￣)，单位是b，100Mb/s=12.5MB/s ，而且一般还达不到……

  • 各层相关协议

  • DNS补充：客户机与服务器之间用 UDP，服务器与服务器之间用 TCP

二、物理层

  1. 传输媒介：网线 / 光纤 / 空气

  2. 信号：模拟信号（用放大器处理，不可复原）

        数字信号（用中继器处理，抗干扰能力强，可复原，可传输距离短）

    光信号：

    光线类型:

    1)单模光纤 一般线是黄色

    2)多模光纤 一般线是橙色/蓝色

  3. 网线 / 双绞线：(双绞，抗干扰，电磁场抵消)

    • 5类双绞线
    • 超5类双绞线
    • 6类 7类 chegn
    • 跳线：比较短的网线，1-3米长

  4. 制作标准：（面向水晶头铜色拨片，颜色线顺序，加粗不变）

    T568A：白绿、绿、白橙、蓝、白、橙、白棕、棕

    T568B：白橙、橙、白绿、蓝、白、绿、白棕、棕

  5. 网线的用途分类： 具有三层功能以上的设备为同种设备

    • 交叉线：一端为 A(发送)，一端为 B(接收)。同种设备间使用！
      如路由器 ↔ 路由器，路由器 ↔ 电脑
    • 直通线：两端都是 A 或 B，目前行业所见都是B线。异种设备间使用！如交换机 ↔ 电脑。
    • 全反线：一端为 A，另一端为反 A，也称 console 线，用于配置企业路由器/交换机

  6. 关于交换机的接口：

    Ethernet 10Mb/s 简写: E
    FastEthernet 100Mb/s F
    GigabitEthernet 1000Mb/s G
    TenGigabitEthernet 10000Mb/s TG

    例如 Ethernet0/3 中，0 是模块编号，3 是该模块的接口编号

  7. 家里一般用的”路由器“其实是交换机和路由器的杂交产品，真正的名字是”三层交换机“
