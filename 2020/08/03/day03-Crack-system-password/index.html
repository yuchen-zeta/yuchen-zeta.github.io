day03 - 破解系统密码

发表于2020-08-03更新于2020-08-03字数统计1k

一、利用5次shift漏洞破解win7密码

  1. 漏洞

    • 漏洞1：未登陆系统前连按5次shift弹出程序（粘滞键）
      程序位置：c:\windows\system32\sethc.exe

    • 漏洞2：部分win7及win10系统在未进入系统时，可以通过系统修复漏洞篡改系统文件名！

    • 注：如果win7或win10已修补此漏洞2，则无法利用

  2. 破解过程相关知识

    • cmd工具路径 c:\windows\system32\cmd
    • 用户/账户密码存储位置 c:\windows\system32\config\SAM
    • 修改账户密码 net user 用户名 新密码

  3. 漏洞利用过程

  • 案例：破解win7系统密码

    • 实验步骤：

    1. 开启win7虚拟机，开机，随便设置一个密码
      可用 net user Barry sad#adhk&*hkadb13asd45 设置

    2. 关机后开机，进入登录页面，鼠标进入虚拟机，连按 shift 5次，弹出对话框，说明存在 漏洞1：5次shift漏洞

    该程序的位置：c:\windows\system32\sethc.exe（突破口：将其替换）

    1. 若存在漏洞，则关机再开机，在出现windows启动界面(四个小点汇聚成旗)时，强制关机(把虚拟机电源拔了，虚拟机右键选择电源—关闭客户机)

    2. 再开机，如果没有出现以下界面，再强制关机一次
      （win7 一般两次，win10 一般三次）
      选择启动修复（漏洞在此哦）


    进入 “ 启动修复 ” 界面

  1. 出现系统还原提示，点击取消，然后它继续尝试修复，等待几分钟

    出现问题原因，如以下界面，点击查看详细信息

  2. 在详细信息中，点击最后一个 txt 文档 的链接，进入记事本

    选择文件 — 另存为 (或打开)，进入以下界面，此时就是在最高权限下查看所有文件

  1. 接下来去寻找 5次shift启动的粘滞键程序

    • 位置： c:\windows\system32\sethc.exe
      将其替换为cmd程序
    • 位置： c:\windows\system32\cmd
      注：此时我们要找的C盘在system权限下其实是D盘
    • 进入D盘，进入windows中的system32中，选择显示所有文件
      找到 sethc ，将其重命名（任意），如sethc-bak
      找到 cmd ，复制一份cmd，改名为sethc
      此时基本已经成功，再次开机使调用sechc时就是调用cmd了

  1. 全部关闭，重新启动，回到登录界面 连按5次shift ，cmd窗口就被调用出来了
    接下来可以为所欲为了，要想进入有以下两种方法
    • 若想修改该用户的密码：
      net user 用户名 新密码
      用户名在登录界面可以看到，若不想设密码，可以输入“”，置空
    • 若不想修改该用户的密码：
      net user test 123 /add #新建用户，设置密码
      net localgroup administrators test /add #升权为管理员
      做完该做的事之后……
      net user test /del #删除用户，擦除数据
      注：还需删除日志，今后会提到，此处暂未提及

二、利用PE破解系统密码

  • Windows PE：Windows Preinstallation Environment，Windows预安装环境，是带有有限服务的最小Win32子系统，基于以保护模式运行的Windows XP Professional及以上内核。
  • 功能：包括运行Windows安装程序及脚本、连接网络共享、自动化基本过程以及执行硬件验证所需的最小功能
  • 使用极其简单：
    • 下一个PE工具，插入U盘，将系统装入，开启目标系统电脑，在刚开机使按相应方法进入BIOS界面(Basic Input Output System,基本输入输出系统)
      • 真实机一般狂摁某个按键，如Esc，F1等
      • 虚拟机，进入下拉菜单，点击 打开电源时进入固件
    • 到BOOT页面下选择U盘或是光驱启动，F10保存，退出BIOS，进入PE系统，点击 破解Windows登录密码
    • 软件自动定位到C:\WINDOWS\SYSTEM32\CONFIG\SAM
      打开后选择用户，更改口令，输入新密码，若不输入，则不设密码
      保存更改，破解成功

原理：SAM保存着所有用户的账号和密码hash值，我们要做的就是进入SAM文件，用 我们要设置的新密码的hash值 替换 针对账号的密码hash值
